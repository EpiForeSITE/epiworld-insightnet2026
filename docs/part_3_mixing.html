Part 3: Mixing Models

Note

This example has been adapted from the ongoing work on the FIFA World Cup, InsightNet’s ID Cup https://github.com/EpiForeSITE/idcup.

Now, we will build a more complex model using the ModelSEIRMixingQuarantine() model. The new model has the following components:

  • Contact Matrix: Using data from Epistorm (Litvinova et al. 2025), this provides a contact matrix at the group level, indicating the average number of contacts between age groups.
  • Heterogenous Vaccination: We can specify vaccination rates to be heterogenous at the group level.
  • Contact tracing and isolation: This is built-in feature in the model. Although users could implement their own using R, it is not recommended as the C++ version is highly efficient.

Preparing the environment

library(epiworldR)
library(measles)
library(data.table)
library(ggplot2)

params <- list(
  city     = "Miami",
  max_pop  = 10000,
  nthreads = 4,
  seed     = 8812,
  ndays    = 60,
  nsims    = 100
)

Pulling information about the city and population from the data files. The mixing matrix is based on the EpiStorm (Litvinova et al. 2025) survey data, which provides age-based contact patterns for various cities. We will use the mixing matrix corresponding to the specified city in params$city.

# Population matrix
census_age <- fread("data/census_age.csv")[city == params$city]
population <- 487014

# Mixing matrix (age-based, from Epistorm survey data)
mixing_matrices <- readRDS("data/mixing_matrix.rds")

The total population in the census data 487014, exceeds the maximum population size (10000). The population will be scaled down to fit within the specified maximum population size.

N <- min(params$max_pop, sum(population))

census_age[, total := ceiling(total / sum(total) * N)]
N <- sum(census_age$total)

We calibrate the contact matrix to a target basic reproduction number of 2.5, consistent with a moderately transmissible respiratory pathogen (e.g., pandemic influenza). The infectious period is set to 7 days and the per-contact transmission probability to 0.1.

calib_scale <- calibrate_mixing_model(
  contact_matrix         = mixing_matrices,
  target_rep_number      = 2,
  infectious_period_days = 7,
  transmission_prob      = 0.1
)

Baseline scenario

# Hospitalization rate such that P(hosp) = 20%
# P(hosp) = hosp_r / (hosp_r + rec_r)
#   => hosp_r = P(hosp) * rec_r / (1 - P(hosp))
h_rate <- 0.2 * (1 / 7) / (1 - 0.2)

seir_model <- ModelSEIRMixingQuarantine(
  name                         = "Respiratory pathogen",
  n                            = N,
  prevalence                   = 10 / N,
  transmission_rate            = 0.1,
  incubation_days              = 5,
  recovery_rate                = 1 / 7,
  contact_matrix               = mixing_matrices * calib_scale,
  hospitalization_rate         = h_rate,
  hospitalization_period       = 7,
  # Detection and intervention parameters
  days_undetected              = 3,
  quarantine_period            = 7,
  quarantine_willingness       = 0.8,
  isolation_willingness        = 0.6,
  isolation_period             = 7,
  # Contact tracing parameters
  contact_tracing_success_rate = 0.5,
  contact_tracing_days_prior   = 4
)
# Adding age-group entities to the model
add_entities_from_dataframe(
  model        = seir_model,
  entities     = census_age,
  col_name     = "age_group",
  col_number   = "total",
  as_proportion = FALSE
)

We will use a single initial infectious case seeded randomly in the population.

get_virus(seir_model, 0L) |>
  set_distribution_virus(
    distfun = distribute_virus_randomly(
      prevalence    = 10,
      as_proportion = FALSE
    )
  )

We can now run the simulations.

saver <- make_saver("outbreak_size", "hospitalizations", "transition")

seir_model |>

    nsims    = params$nsims,
    seed     = params$seed,
    saver    = saver,
    nthreads = 1 # params$nthreads
  )

Intervention: reducing contact rates by 75%

Interventions in epiworldR can be applied in various forms. Besides changing the initial conditions, we can change the situation midway through the simulation using global events. Global events are functions that can arbitrarily modify the model at user-defined points in time. Some of the common applications are:

  • globalevent_set_params(): Change model parameters (e.g., transmission rate, recovery rate, etc.) at a specific day.
  • globalevent_tool(): Distribute a tool (e.g., vaccine, antiviral, etc.) to the population at a specific day.
  • globalevent_fun(): Apply an arbitrary R function to the model at a specific day. This is the most flexible option, as it allows users to implement any intervention that can be coded in R. However, it is also the least efficient option, as it requires running the simulation in a single thread.
WarningThread safety

Unfortunately, the R programming language is not thread safe. In other words, when we use the globalevent_fun() caller, which essentially wraps an R function, we need to run the run_multiple() function using a single thread.

Global events can, as I mentioned, modify arbitrary parts of the program. In this case, we will use a global event to reduce the contact rates by 75% starting from day 40. This is equivalent to reducing the contact matrix by a factor of 4. To do so, we need two steps:

# Step 1: Create a R function that receives a model
reduce_contact_fourth <- function(m) {

  cmat <- epiworldR::get_contact_matrix(m)
  epiworldR::set_contact_matrix(
    m, cmat / 4,
    as_backup = FALSE
  )

}

# Step 2: Register (add) the event to the model
add_globalevent(
  model = seir_model,
  event = globalevent_fun(
    fun  = reduce_contact_fourth,
    name = "Reduce contact",
    day  = 40
    )
  )

The as_backup = FALSE tells epiworldR to automatically restore the original contact matrix when restarting the simulation. Let’s see how the model changes with this new event.

saver2 <- make_saver("outbreak_size")

seir_model |>
  run_multiple(
    ndays    = params$ndays,
    nsims    = params$nsims,
    seed     = params$seed,
    saver    = saver2,
    nthreads = 1 # params$nthreads
  )
